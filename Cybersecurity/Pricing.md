
## 1. เลือกโมเดลการคิดราคา (Pricing Models)

### แบบที่ 1: คิดตามขอบเขตงาน (Fixed Scope / Asset-Based)

วิธีนี้เหมาะสำหรับงานที่มีโครงสร้างชัดเจน ลูกค้าทราบจำนวน Assets ที่แน่นอน เป็นวิธีที่เข้าใจง่ายที่สุดทั้งสองฝ่าย

* **Web Application:** นิยมคิดตามจำนวนความซับซ้อน, จำนวนหน้า (Static vs Dynamic), หน้า Login และจำนวน User Roles/Permissions (เช่น สิทธิ์ Admin, Staff, User ยิ่งสิทธิ์เยอะราคายิ่งสูง)
* *เรทตลาดคร่าวๆ:* ระบบขนาดเล็ก-กลาง (1-2 Roles) เริ่มต้นตั้งแต่ **฿50,000 – ฿150,000+** หากเป็นระดับ Enterprise หรือ Fintech ที่มี Logic ซับซ้อนและมี API ต่อพ่วงจำนวนมาก ราคาอาจขยับไปถึง **฿300,000 – ฿600,000+**

* **Mobile Application:** คิดแยกตาม OS (iOS / Android) พ่วงด้วยความซับซ้อนของ API Backend
* *เรทตลาดคร่าวๆ:* แพลตฟอร์มเดี่ยวเริ่มต้นประมาณ **฿100,000+** ถ้าควบทั้งสองแพลตฟอร์มพร้อม Backend มักจะอยู่ที่ **฿200,000 – ฿400,000+**

* **Network Infrastructure (Internal / External):** มักคิดตามจำนวน IP Address หรือ Live Hosts
* *เรทตลาดคร่าวๆ:* โครงสร้างพื้นฐานขนาดเล็กหรือ External IP ไม่กี่เบอร์ อยู่ที่ช่วง **฿50,000 – ฿150,000** แต่ถ้าเป็น Internal Network ขนาดใหญ่ที่มีการทดสอบ Active Directory (AD), Lateral Movement หรือการทำ Network Segmentation ก็จะขยับไปที่ **฿250,000++**



### แบบที่ 2: คิดตามเวลาที่ใช้จริง (Man-Day Rate)

เหมาะสำหรับงานที่ขอบเขตไม่ชัดเจน หรือเป็นงานประเภท Red Teaming / Cloud Environment ที่ต้องการความลึกในการงัดแงะ

* **สูตรคำนวณ:** `ราคา = (จำนวนคน x จำนวนวัน) x Man-Day Rate`
* **Man-Day Rate ในไทย:** * **Freelance / Junior-Mid Level:** ฿8,000 – ฿15,000 ต่อวัน
* **Senior / Specialist (มี Cert ระดับสูง เช่น OSCP+, OSCE, GXPN):** ฿18,000 – ฿35,000+ ต่อวัน
* *ตัวอย่าง:* งาน Web App ขนาดกลาง ใช้เวลาแฮกและเขียนรายงาน 10 วัน โดย Senior 1 คน = $10 \times 20,000 = \text{฿}200,000$

## 2. 4 ปัจจัยสำคัญที่ต้องบวกเพิ่ม (Cost Variables)

ก่อนส่งใบเสนอราคา (Quotation) ควรเช็กปัจจัยเหล่านี้เพื่อไม่ให้เข้าเนื้อตัวเอง:

* **1. รูปแบบการทดสอบ (Black, Gray, or White Box):**
* *Black-box* อาจใช้เวลาลองผิดลองถูกช่วงแรกเยอะ
* *Gray/White-box* มีการดู Source Code หรือสแกนควบคู่ไปด้วย แม้จะเจอช่องโหว่ลึกและตรงจุดกว่า แต่ผู้ทดสอบต้องใช้แรงและเวลาในการรีวิวโค้ดสูงขึ้น ราคาจึงมักจะแพงกว่า Black-box ทั่วไป


* **2. ข้อกำหนดด้าน Compliance & Regulations:**
* หากลูกค้าต้องการรายงานที่ต้อง Map เข้ากับมาตรฐานเฉพาะ เช่น **PCI DSS, ISO 27001, PDPA** หรือเกณฑ์ของ **ธปท. / กลต.** คุณจะต้องใช้เวลาในการทำเอกสารและ Mapping Control เพิ่มขึ้นอย่างมาก ควรบวกค่าจัดการเอกสารส่วนนี้เพิ่ม


* **3. การทดสอบซ้ำ (Retest / Verification Scan):**
* ปกติแล้วควรระบุให้ชัดเจนว่าราคานี้ **"รวม Retest 1 รอบฟรี ภายใน 30-90 วันหลังจากส่งรายงานรอบแรก"** เพื่อให้ลูกค้าไปอุดช่องโหว่แล้วเรากลับมาตรวจซ้ำ หากลูกค้าต้องการ Retest มากกว่า 1 รอบ ควรคิดเงินเพิ่มเป็นสัดส่วนต่อรอบ


* **4. สถานที่ปฏิบัติงาน (On-site vs Remote):**
* หากต้องเข้าไปนั่งทำที่หน้างาน (เช่น ตรวจ Internal Network หรือตู้คีออสที่เชื่อมสายภายนอกไม่ได้) ต้องคำนวณค่าเดินทาง ค่าที่พัก และค่าเสียเวลาเดินทางบวกเข้าไปด้วย

## 3. สิ่งสำคัญที่ต้องระบุในขอบเขตงาน (Scope Definition)

เพื่อป้องกันปัญหา "งานงอก" หรือ Scope Creep ภายหลัง ในสัญญาหรือใบเสนอราคาต้องระบุให้ชัดเจน:

1. **Exclusion:** อะไรที่ห้ามทำ (เช่น ห้ามทำ DoS/DDoS, ห้ามยุ่งกับ Production DB ที่ส่งผลกระทบต่อระบบ)
2. **Reporting:** ระบุว่ารายงานจะมี 2 ส่วนเสมอ คือ *Executive Summary* (สำหรับผู้บริหาร อ่านง่าย มีกราฟสรุปความเสี่ยง) และ *Technical Report* (สำหรับ Developer พร้อมวิธีแก้ไขและหลักฐาน Proof of Concept)
3. **Timeframe:** ระบุวันเริ่มเทส วันส่งรายงานฉบับร่าง (Draft Report) และกรอบเวลาในการทำ Retest ให้ชัดเจน

หากเพิ่งเริ่มต้นรับงานเอง แนะนำให้เริ่มจากการประเมินแบบ **Man-Day** เพื่อดูว่าตัวเองต้องใช้เวลาแกะระบบนั้นๆ นานแค่ไหน จากนั้นลองนำมาเปรียบเทียบกับราคาตลาดในกลุ่ม Asset ใกล้เคียง เพื่อให้ได้ราคาที่ไม่ตัดราคาตลาดจนน่าเกลียด และสะท้อนถึงทักษะและความละเอียดของงานที่เรามอบให้ลูกค้าครับ


ค่าบริการจะขึ้นอยู่กับค่าซับซ้อนของ website หรือ application ครับ
สำหรับงานนี้จะเริ่มต้นที่ประมาณ 15,000 บาทครับ

ผมขอสอบถามเพิ่มเติม
- _ถ้าเป็น Web/API:_ มีจำนวนกี่หน้า หรือมีกี่ Endpoints/APIs? มีหน้าบ้าน-หลังบ้านแยกกันมั้ย?
- _ถ้าเป็น Mobile App:_ มีระบบปฏิบัติการอะไรบ้าง (iOS, Android หรือทั้งคู่)?
- การทำ Pentest ครั้งนี้จุดประสงค์หลักเพื่ออะไร (เช่น ตรวจสอบความปลอดภัยประจำปี, มีระบบเพิ่งพัฒนาเสร็จใหม่, หรือต้องการส่งรายงานให้ Partner/ลูกค้าตรวจเช็ก, ต้องการมาตราฐาน PCI DSS)
- ระบบมีความเกี่ยวข้องการระบบการเงินอะไรบ้าง
